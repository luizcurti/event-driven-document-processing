Document Processing Platform

Imagine a system similar to those used by Uber, banks, or HR platforms.

A user uploads a PDF document.

The system then:

Stores the file in Amazon S3
Creates an event
Starts an AWS Step Functions workflow
Extracts document information
Generates thumbnails
Performs OCR
Stores metadata
Sends notifications
Records all operations in Amazon CloudWatch

Development Guidelines

Apply SOLID, KISS, and YAGNI principles where appropriate.
Follow Clean Code best practices.
Use DDD and Clean Architecture only when they add clear value; avoid overengineering.
Favor simple, readable, and maintainable code.
Write unit and integration tests with the goal of achieving 100% code coverage.
